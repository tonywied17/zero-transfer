## What's new

- **feat(providers)**: add optional `remove`, `rename`, `mkdir`, `rmdir` methods to `RemoteFileSystem`. Implemented for FTP, FTPS (via shared `FtpFileSystem`), SFTP, Local, and Memory providers.
- New public option types: `RemoveOptions`, `RenameOptions`, `MkdirOptions`, `RmdirOptions`.
- `mkdir` and `rmdir` support a `recursive` flag; `remove`/`rmdir` support `ignoreMissing`.

## Provider coverage

| Provider | remove | rename | mkdir | rmdir |
| --- | --- | --- | --- | --- |
| `@zero-transfer/ftp`, `@zero-transfer/ftps` | DELE | RNFR/RNTO | MKD (recursive supported) | RMD (recursive walks children) |
| `@zero-transfer/sftp` | unlink | rename | mkdir (recursive supported) | rmdir (recursive walks children) |
| `@zero-transfer/core` (local) | fs.unlink | fs.rename | fs.mkdir | fs.rm |
| memory provider | in-map | in-map | in-map | in-map |

Cloud providers (S3, Azure Blob, GCS, Dropbox, Google Drive, OneDrive, HTTP, WebDAV) do not yet implement these methods. Consumers can feature-detect via `typeof session.fs.remove === 'function'`.
